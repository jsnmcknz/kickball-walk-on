S1.6 appearance ground truth. Tap-to-draft (Jason, 2026-07-11): the order is built by tapping roster tiles — each tap appends the player to the bottom of the list and removes their tile. ✕ on a row returns the player to the grid; drag handles reorder (vertical drag only — simplifies the drag math behind game-1 defect #2); row 3 shows mid-drag treatment (moss border + lift). Anyone left in the grid at start simply isn’t kicking tonight. “+ new” = name → member/sub → pick a defaultClips song → player_add event, then their tile appears (or lands straight in the order — TBD on feel). Opponent comes from the fixture, never typed; change game lists other fixtures + an “unscheduled game” fallback (allowed to be ugly). Start appends game_start with scheduleGameId and enters the game screen.

The three S1.6 sheets that had no visual anywhere (the "small and unmocked breeds generic UI" guard). PIN gate: in-app keypad, not a system text field — no keyboard popping over a kiosk screen; dots fill as digits land; wrong PIN shakes and clears (no error prose); once per device, persisted. Add player: the one place typing is unavoidable (system keyboard OK, it's a rare pre-game op); member/sub segment; song list = the manifest's defaultClips with durations, tap to preview via the normal engine; primary action reads add to order because the new player lands straight in the order (07's open question, resolved by this sheet's copy — flip on preview if it feels wrong). End game: plum = "this ends" semantic, echoes the score for confirmation, cancel is explicit about resuming. All three render as takeover sheets in the app's existing sheet conventions, not modals over dimmed content.

Spec S1 item 3's "tappable figures → adjust steppers", finally drawn (confirmed IN by Jason 2026-07-11; layout per his revisions — OUTS + INNING box on top, SCORES box below it with team names above side-by-side steppers). ONE tap anywhere on the score bug opens the whole sheet — per-figure tap targets inside that dense bug would be fiddly at the park. Stepper squares 32px to fit two abreast inside the boxes. Draft semantics (Jason's revision, superseding the instant-log version): steppers edit a local draft; done commits — one `adjust` event per changed field; cancel discards everything, even after changes. Cleaner event log too (one adjust per field instead of one per tap). Inning stepper flips through halves and shows arrow + bare numeral for space (▲3 → ▼3 → ▲4 — Jason 2026-07-11; ordinals stay everywhere else); outs clamp 0–2 (a third out is a real event, not an adjustment). Team shorthand is the manifest's new `teamShort` field ("PICKLES", replacing the initials-derived PPP app-wide). Renders in the correction-sheet slot on the offense screen, same dim-the-rest treatment as the runner sheet.
